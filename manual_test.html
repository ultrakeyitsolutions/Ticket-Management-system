Manual Test Instructions

Steps to test the remove functionality:

  1. Navigate to: Admin Profile Page
  2. Login with admin credentials
  3. Upload a profile picture first (if you don't have one)
  4. Click the "Remove" button next to your profile picture
  5. Confirm the removal in the dialog
  6. The page should reload and the profile picture should be gone

What was fixed:

  • Added remove_profile_picture action handler in the backend view
  • Added proper profile picture deletion logic
  • Added handling for address fields (city, state, country, address)
  • The frontend JavaScript was already correctly implemented

Technical Details:

The issue was in apps/dashboards/views.py in the admin_dashboard_page function. The view was missing the handler for the remove_profile_picture action that the frontend JavaScript was sending.

Before: The remove button would submit the form but the backend would ignore the action, so nothing would happen.

After: The backend now properly handles the remove_profile_picture action, deletes the profile picture file, sets the field to None, and saves the profile.
